Discover and execute API operations.

Your agent stays in control.

YellowPages is an MCP server that exposes two tools: discover (RAG over an OpenAPI schema, returns matching operations with their schema) and execute (runs an operation by ID with given params). No LLM inside the server—your caller agent chooses which operation and parameters to use.

Go to docs How it works

Why it exists

APIs are everywhere, but using them usually means reading specs, copying URLs, and wiring parameters. YellowPages inverts that: you load an OpenAPI schema once and index it, then your MCP host (e.g. Cursor) can call discover with a natural-language query to get matching operations (with IDs and JSON schema), then call execute with the chosen operation and params. The server does RAG for discovery and runs the HTTP call; the caller agent decides what to call and with what arguments.

How it works

Build the index once; then the caller uses discover then execute. The server expects the Chroma directory to already exist.

  1. 1

    Indexer

    Build the vector DB once (or when the schema changes). Run the indexer with your OpenAPI schema path and Chroma directory. It flattens the schema into operations and indexes them in Chroma with local embeddings (sentence-transformers/all-MiniLM-L6-v2 by default).

  2. 2

    Discover

    The MCP server exposes discover_operations: RAG only; returns matching operations with ID and schema. Your agent sends a natural-language query and gets back operations with name, method, url, and parameters.

  3. 3

    Execute

    The server exposes execute_operation: runs an operation by name with path/query/body params. No LLM inside the server; the caller agent chooses what to execute.

Get started

Add YellowPages to your MCP host and point it at your OpenAPI schema.

Quick start

uv sync
uv run python -m src.indexer
uv run main.py

Build the vector index from your OpenAPI schema (once, or when schema changes), then start the MCP server. No API key needed; uses local embeddings.

Adding to Cursor

In MCP settings, add a server entry (UV recommended):

{
  "mcpServers": {
    "yellow-pages": {
      "command": "uv",
      "args": ["run", "/path/to/yellow-pages/main.py"]
    }
  }
}

Use discover_operations with a natural-language query, then execute_operation with the chosen operation name and params. Full README and environment variables are in the repo.
